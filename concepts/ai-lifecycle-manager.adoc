[#ai-lifecycle-manager-roles]
== Workflows and responsibilities

The workflow is defined by a user role and is different for a user and an administrator.

Administrator::
  * Installs {sailifecyclemanager}.
  * Discovers, deploys and manages AI applications from {suse} trusted registries such as {sappco} and {sregistry}.
  * Manages users and user roles.
  * creates ClusterRepo repositories.
User::
  * Installs AI applications using the {sailifecyclemanager}.
  * Manages application instances

[#ai-lifecycle-manager-benefits]
== Benefits compared to manual deployment

Centralized, role-based control::
  Permissions are enforced inside {ranchera} and split into roles.
  This approach offers greater security and governance without giving everyone {helm} access.

  Installed and used entirely from the {ranchera} Dashboard.

Curated setup for AI workloads::
  Encodes best practices for AI applications, it assumes:
  * private registries
  * controlled deployment paths
  * abstracts complexity away from end users

  * centrally configured
